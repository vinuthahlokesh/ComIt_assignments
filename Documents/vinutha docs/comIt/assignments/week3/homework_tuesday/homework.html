What are the primitive types in JS ? Give example of each.

In JavaScript, the primitive types are:

  • String: Represents a sequence of characters. Example: "Hello, World!"
  • Number: Represents both integer and floating-point numbers. Example: 42 or 3.14
  • Boolean: Represents a logical entity and can have two values: true or false.
  • Undefined: A variable that has been declared but not assigned a value. Example: let x;
  • Null: Represents the intentional absence of any object value. Example: null
  • Symbol: A unique and immutable value often used as an identifier for object properties. Example: Symbol('description')
  • BigInt: An arbitrary-precision integer type that can represent integers larger than the Number type can hold. Example: 1234567890123456789012345678901234567890n

These primitive types are immutable and represent single values, unlike objects which can hold collections of values and more complex entities.

When should you use an object

In JavaScript, you should use an object when you need to:

  • Group related data together: Objects allow you to store multiple values in a single entity, making it easier to manage and access related information.
  • Represent complex entities: When you need to model real-world objects with properties and behaviors, objects are the way to go.
  • Use key-value pairs: Objects allow you to use strings (or Symbols) as keys to access values, providing a flexible way to organize data.
  • Implement methods: Objects can contain functions as properties, allowing you to define behaviors associated with the data they hold.
  • Maintain state: Objects can hold and manage state over time, making them suitable for applications that require data persistence.

In contrast, primitive types are best used for simple values that do not require additional structure or behavior.

When should you use an array

In JavaScript, you should use an array when you need to:

  • Store a list of items: Arrays are ideal for holding collections of data, such as numbers, strings, or objects, in a sequential manner.
  • Maintain order: Arrays preserve the order of elements, allowing you to access items by their index.
  • Perform operations on collections: Arrays come with built-in methods for iterating, filtering, mapping, and reducing data, making it easier to manipulate collections.
  • Handle dynamic sizes: Arrays can grow and shrink in size as needed, making them flexible for various use cases.
  • Use array-specific methods: Arrays have methods like push, pop, shift, and unshift for adding or removing elements, which are not available in objects.

In contrast, objects are better suited for representing entities with properties and methods, while arrays are designed for ordered collections of items.
